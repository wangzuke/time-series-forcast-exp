### 3.3 主实验关键发现

1. **最简单的方法最强**：Interp+iTransformer 以平均 MSE 0.3264（10% 缺失）/ 0.3411（30% 缺失）取得全局最优，退化率甚至为负值（-5.6% / -1.4%），意味着线性插值填补 + iTransformer 在缺失条件下的表现接近甚至略优于无缺失基准平均。

2. **预测模型的选择比缺失处理策略更重要**：无论使用线性插值还是 SAITS 补值，iTransformer 始终优于 PatchTST，PatchTST 始终优于 DLinear。DLinear 系列（Interp+DLinear、SAITS+DLinear）退化率高达 +13~22%，而同一缺失处理下 iTransformer 退化率接近 0。

3. **缺失感知方法未能胜过两阶段方法**：三个缺失感知方法（MissTSM、CRIB、CoIFNet）的退化率均显著高于 Interp+iTransformer 和 SAITS+iTransformer。其中 MissTSM 退化率最高（+27%），CoIFNet 在缺失感知方法中表现最好但仍然落后于最佳两阶段方法。

4. **数据集特征决定方法排序**：在 ETTh1（低维、规律性强）上，线性插值 + 简单模型即可近似无缺失性能；在 Traffic（高维、862 变量）上，只有 iTransformer 能有效利用跨变量依赖，DLinear 退化严重。

## 六、研究问题回答

本节逐一回答实验计划第 4 节提出的六个研究问题，对照实验假设，基于实验数据给出结论。

### 问题一：缺失率升高时，不同方法的预测误差如何变化？

> **实验假设**：缺失率越高，传统两阶段方法的误差增长越明显；直接利用部分观测序列的方法退化较慢。

**实验结论：假设部分成立，但退化模式与预期不同。**

从鲁棒性分析表中可以看到，当缺失率从 10% 升高到 30% 时：

| 方法类别 | 10% → 30% MSE 变化 | 退化率增幅 |
|---|---|---|
| Interp+iTransformer（两阶段简单填补） | 0.3264 → 0.3411 | +4.5% |
| SAITS+iTransformer（两阶段强补值） | 0.3337 → 0.3506 | +5.1% |
| CoIFNet（联合补值预测） | 0.3755 → 0.3831 | +2.0% |
| CRIB（缺失感知） | 0.3872 → 0.3944 | +1.9% |
| MissTSM（不补值直接预测） | 0.4418 → 0.4405 | -0.3% |

缺失感知方法（MissTSM、CRIB、CoIFNet）确实对缺失率变化更鲁棒（增幅 < 2%），而两阶段方法增幅约 4~5%。但是，**两阶段方法在两个缺失率下的绝对性能均优于缺失感知方法**。MissTSM 虽然几乎不随缺失率变化（退化率从 27.7% 到 27.4%），但其起点就远高于两阶段方法。

换言之，缺失感知方法的"鲁棒性"更多源于其本身性能较差导致的低基线效应，而非真正的缺失适应能力。

### 问题二：随机点缺失和连续片段缺失对模型的影响是否相同？

> **实验假设**：连续片段缺失对预测的破坏更大，因为连续缺失会破坏局部时间变化模式。

**实验结论：假设整体成立，但差异幅度因方法和数据集而异。**

![随机点 vs 连续片段](../figures_vis/bar_rp_vs_cs.png)

以 30% 缺失率、预测长度 96 为例，比较各方法在两种缺失下的 MSE：

| 方法 | ETTh1 RP→CS | Weather RP→CS | Electricity RP→CS | Traffic RP→CS |
|---|---|---|---|---|
| Interp+iTrans | 0.4010→0.4373 (+9.0%) | 0.1742→0.1876 (+7.7%) | 0.1636→0.1860 (+13.7%) | 0.4373→0.4912 (+12.3%) |
| SAITS+iTrans | 0.4319→0.4230 (-2.1%) | 0.1779→0.1788 (+0.5%) | 0.1608→0.1630 (+1.4%) | 0.4374→0.4515 (+3.2%) |
| CoIFNet | 0.4076→0.4184 (+2.6%) | 0.1631→0.1630 (-0.1%) | 0.1766→0.1924 (+8.9%) | 0.6096→0.6528 (+7.1%) |

对于简单填补方法（Interp），连续片段缺失的 MSE 普遍高于随机点缺失 7~14%，符合假设。但对于 SAITS 两阶段方法，差异很小（1~3%），说明 SAITS 的补值能力在一定程度上弥补了连续缺失的信息损失。

从缺失类型泛化实验（4.3 节）也能看到：仅在随机点缺失上训练的 MissTSM 迁移到连续片段缺失时退化仅 6~9%，但迁移到变量通道缺失时退化高达 32~68%。这表明**变量通道缺失是破坏性最大的缺失模式**，因为它导致某些变量在整个输入窗口内完全不可见，破坏了跨变量依赖关系。

### 问题三：补值效果好是否一定意味着预测效果好？

> **实验假设**：补值误差低不一定代表预测误差低，因为补值任务和预测任务的优化目标不同。

**实验结论：假设成立。补值 MSE 与预测 MSE 之间不存在正相关关系。**

从补值误差传播实验（4.2 节）中可以提取如下关键对比：

**典型案例 1——ETTh1 随机点缺失 10%**：
- 线性插值：补值 MSE = 0.1060，预测 MSE = 0.4250
- SAITS：补值 MSE = 3.6544，预测 MSE = 0.5228

SAITS 的补值误差是线性插值的 **34 倍**，但预测误差差距仅 23%。

**典型案例 2——Electricity 连续片段缺失 10%**：
- 线性插值：补值 MSE = 1.3179，预测 MSE = 0.1757
- SAITS：补值 MSE = 0.2781，预测 MSE = 0.1690

SAITS 补值 MSE 仅为线性插值的 **1/5**，但预测 MSE 仅好了 3.8%。

**典型案例 3——Weather 随机点缺失 10%**：
- 线性插值：补值 MSE = 0.0392，预测 MSE = 0.2089
- SAITS：补值 MSE = 3.0451，预测 MSE = 0.2336

线性插值的补值 MSE 极低（比 SAITS 低 78 倍），但预测性能反而更好。

这些结果证实了 CRIB [6] 和 CoIFNet [7] 论文的核心论点：补值任务追求的是在缺失位置恢复原始值，而预测任务关心的是利用历史信息预测未来。两个优化目标并不完全一致，因此"补值更好 → 预测更好"的逻辑链并不成立。

特别值得注意的是，在随机点缺失场景中，线性插值的补值精度很高（补值 MSE < 0.15），因为邻近时间步通常有观测值可供插值；但在连续片段缺失场景中，线性插值的补值精度大幅下降（补值 MSE 可达 1.0~2.3），而 SAITS 由于能利用跨变量信息，补值精度更好。尽管如此，**预测性能上两者差距很小**，再次说明下游预测器能够一定程度上适应补值噪声。

### 问题四：缺失掩码是否能够提升预测效果？

> **实验假设**：加入缺失掩码后，模型能够区分真实值与缺失位置，预测性能更稳定。

**实验结论：假设不成立。在当前实验设置下，加入缺失掩码反而损害了预测性能。**

缺失掩码消融实验结果：

| 输入形式 | 全局平均 MSE | 对比 A 的增幅 |
|---|---:|---:|
| A: 只输入数值 | 0.3020 | — |
| B: 数值 + 掩码 | 0.4117 | +36.3% |
| C: 数值 + 掩码 + 时间 | 0.3527 | +16.8% |

形式 A（不提供掩码）在全局平均和绝大多数单个配置中都取得了最好的 MSE。加入掩码后性能不升反降，且降幅很大。这一结果与假设完全相反。

可能的解释：

1. **额外输入维度增加了模型学习难度**：掩码和时间位置作为额外通道拼接到输入中，使得输入维度翻倍甚至三倍。在 10 epoch 的短训练下，模型可能没有足够的容量和训练时间来学会有效利用这些额外信息。

2. **掩码可能引入了错误的注意力偏移**：掩码通道本身具有高方差（0/1 二值分布），可能在训练中分散了模型对有效数值特征的注意力。

3. **缺失感知方法的实现方式差异**：MissTSM、CRIB、CoIFNet 三个缺失感知方法本质上就是在使用掩码信息，但它们的 MSE 也普遍高于不使用掩码的简单方法。这表明本实验中掩码信息的利用方式——无论是拼接输入还是在注意力中使用——都未能带来收益。

4. **实验中三种形式的映射**：A 对应的是线性插值 + iTransformer（不提供掩码），B 对应 MissTSM（掩码参与注意力），C 对应 CRIB（掩码 + 时间位置）。因此这个消融实际上也反映了三种方法本身的能力差异。

需要注意，这个结论可能受限于当前的实验规模（10 epoch）和模型架构。在更长训练、更大模型或不同的掩码融合方式下，结论可能不同。

### 问题五：联合补值预测是否优于先补值再预测？

> **实验假设**：在高缺失率条件下，联合建模方法优于分离式两阶段方法。

**实验结论：假设不成立。在本实验的所有配置中，两阶段方法的最优组合均优于联合建模方法。**

以 30% 缺失率（"高缺失率"条件）下各方法的平均 MSE 为参照：

| 方法类别 | 方法 | 30% 缺失平均 MSE |
|---|---|---:|
| 两阶段（简单填补） | Interp+iTransformer | **0.3411** |
| 两阶段（强补值） | SAITS+iTransformer | 0.3506 |
| 联合补值预测 | CoIFNet | 0.3831 |
| 缺失感知 | CRIB | 0.3944 |
| 不补值直接预测 | MissTSM | 0.4405 |

最优两阶段方法（Interp+iTrans）比最优联合建模方法（CoIFNet）低 12.3%。即使在 SAITS 的有效配置中（排除溢出），SAITS+iTrans 也优于 CoIFNet。

这一结论可能有以下原因：

1. **预测模型的先天优势**：iTransformer 经过大量时间序列预测研究的打磨，其倒置 Transformer 架构在捕捉跨变量依赖方面有成熟优势。而 CoIFNet、CRIB 等方法需要同时解决补值和预测两个任务，在固定训练预算下可能难以同时优化好两个目标。

2. **训练资源限制**：本实验统一使用 10 epoch 训练，对于联合建模方法可能不够充分。两阶段方法的预测模型只需学习"从完整序列到未来预测"的映射，学习负担较轻。

3. **实现简化的影响**：本实验中 CRIB 和 CoIFNet 的实现为简化版本（见 CLAUDE.md），可能未完全还原原论文的完整能力。

### 问题六：简单线性模型在缺失场景下是否仍具有竞争力？

> **实验假设**：在部分规律性较强的数据集上，简单线性模型可能接近复杂模型，因此应作为必要基线。

**实验结论：假设成立，但竞争力高度依赖数据集特征。**

从预测模型复杂度消融中可以看到：

| 数据集 | DLinear vs iTransformer（Interp 填补, 30%, h96） | DLinear 差距 |
|---|---|---|
| ETTh1 | 0.3995 vs 0.4010 | DLinear **更优** 0.4% |
| Weather | 0.2025 vs 0.1742 | 落后 16.2% |
| Electricity | 0.2079 vs 0.1636 | 落后 27.1% |
| Traffic | 0.6816 vs 0.4373 | 落后 55.9% |

在 ETTh1 上，DLinear 与 iTransformer 几乎持平甚至更优，这是因为 ETTh1 只有 7 个变量、周期性明显，线性模型足以捕获主要模式。但在高维数据集上，DLinear 大幅落后：

- **Traffic（862 变量）**：DLinear 的 MSE 是 iTransformer 的 1.56 倍，差距达 56%。高维数据中跨变量依赖关系复杂，线性投影无法有效建模。
- **Electricity（321 变量）**：差距 27%，原因类似。
- **Weather（21 变量）**：差距 16%，变量间的物理关联（温度-气压-风速等）需要非线性建模。

因此，DLinear 作为基线确实是必要的——它帮助我们判断特定数据集的缺失预测问题是否主要由简单趋势驱动。当 DLinear 已经接近最优（如 ETTh1），说明该场景下投入复杂模型的边际收益很小。

## 七、综合结论

### 7.1 核心发现总结

1. **"线性插值 + 强预测模型"是最有效的缺失处理策略**。Interp+iTransformer 在绝大多数配置中取得最优或接近最优的预测性能，退化率甚至为负值，说明简单填补已经足够。

2. **预测模型的选择比缺失处理策略更重要**。同一缺失处理下，从 DLinear 换成 iTransformer 带来的 MSE 降低（18~56%）远大于从线性插值换成 SAITS 带来的变化（<5%）。

3. **缺失感知方法在本实验中未能展现优势**。MissTSM、CRIB、CoIFNet 在所有主实验配置中均不如最佳两阶段方法，退化率也更高。这可能与实现简化、训练轮数有限、以及这些方法原始论文中使用的实验设置差异有关。

4. **补值质量与预测质量不完全挂钩**。SAITS 在补值任务上远优于线性插值，但这并未转化为一致的预测优势，印证了补值-预测目标不一致的理论。

5. **缺失掩码在当前设置下未带来收益**。不提供掩码的模型反而表现最好，但这可能与训练时长不足和掩码融合方式有关。

6. **连续片段缺失确实比随机点缺失更具破坏性**，但差距可控（7~14%）。**变量通道缺失是最具破坏性的缺失模式**，泛化到该模式时退化可达 32~68%。

7. **SAITS 两阶段方法存在数值稳定性问题**。SAITS + PatchTST 组合在 102/288 条实验中发生数值溢出，特别是在连续片段缺失和变量通道缺失场景中。

### 7.2 假设验证汇总

| 研究问题 | 实验假设 | 验证结果 |
|---|---|---|
| 问题一：缺失率对误差的影响 | 两阶段方法退化更明显，缺失感知方法退化更慢 | **部分成立**：缺失感知方法确实退化更慢，但其绝对性能始终劣于两阶段方法 |
| 问题二：缺失类型的影响 | 连续片段缺失破坏更大 | **成立**：连续片段缺失 MSE 高 7~14%；变量通道缺失破坏最大 |
| 问题三：补值好 = 预测好？ | 补值误差低不等于预测误差低 | **成立**：多个案例中补值 MSE 差距数十倍但预测 MSE 差距仅数个百分点 |
| 问题四：缺失掩码的作用 | 掩码能提升性能 | **不成立**：加入掩码后 MSE 增加 17~36% |
| 问题五：联合建模 vs 两阶段 | 联合建模在高缺失率下更优 | **不成立**：最优两阶段方法在所有缺失率下均优于联合建模 |
| 问题六：简单模型的竞争力 | 简单线性模型在部分数据集上有竞争力 | **成立**：ETTh1 上 DLinear 接近 iTransformer，但高维数据集上落后 27~56% |

### 7.3 对实验假设不成立的讨论

问题四和问题五的假设未被验证，可能的原因包括：

1. **训练预算限制**：统一使用 10 epoch + early stopping，缺失感知方法和联合建模方法可能需要更长的训练来收敛。

2. **实现简化**：CRIB 使用简化版信息瓶颈（未实现原文 patching + 双重 ELBO），CoIFNet 使用 TSMixer 风格 backbone（原文架构更复杂）。这些简化可能削弱了方法的实际能力。

3. **超参数统一**：所有方法使用相同学习率（1e-3）、batch size（32），未针对各方法单独调参，可能不利于需要特定超参数的方法。

4. **缺失率范围**：本实验覆盖 10% 和 30% 缺失率。在更极端的缺失条件（如 50%、70%）下，缺失感知方法可能展现更大优势。

这些因素需要在后续实验中进一步验证。
