> 承接《实验计划 2.0》和《0618 实验报告》，本文档规划第二轮实验。

## 1. 第一轮实验回顾

第一轮共完成 2292 组实验（2133 个有效结果），核心发现如下：

| 编号 | 发现 | 说明 |
|---|---|---|
| F1 | 线性插值 + iTransformer 全局最优 | 退化率甚至为负（-5.6% / -1.4%），即缺失反而略提升 |
| F2 | 预测模型选择 > 缺失处理策略 | DLinear→iTransformer 改善 18~56%；线性插值→SAITS 改善 <5% |
| F3 | 缺失感知方法全面落后 | MissTSM 退化率 +27%，CoIFNet +9~11%，CRIB +12~14% |
| F4 | 补值好 ≠ 预测好 | SAITS 补值 MSE 远优于线性插值，但预测 MSE 差距很小 |
| F5 | 加入掩码反而损害性能 | 不提供掩码的模型 MSE 最低，加掩码后 +17~36% |
| F6 | 连续片段缺失比随机点缺失更具破坏性 | MSE 增加 7~14%；变量通道缺失最严重（+32~68%） |
| F7 | SAITS+PatchTST 数值溢出 | 102/288 条失败，集中在 Weather 连续片段缺失和扩展数据集 |

## 2. 第二轮实验动机

第一轮结论 F3（缺失感知方法全面落后）和 F5（掩码无收益）与原始论文的声明相矛盾。经排查，原因可能来自：

### 2.1 模型实现简化

| 方法 | 简化程度 | 缺失的关键特性 |
|---|---|---|
| CRIB | **重** | 缺少 patching 机制（输入应先分 patch 再编码）；缺少 TCNBlock 时序嵌入（用简单 Linear 替代）；缺少双视图一致性正则（原文对同一输入生成两个视图计算一致性损失）；缺少 MultivariateNormal 后验（用简化重参数化替代） |
| CoIFNet | **中** | backbone 固定为双 FFN MixerBlock（原文支持 TSBlock/LinearBlock/AttentionBlock 可配置）；缺少 AttentionBlock 的 Gated Values 机制；缺少时间特征嵌入（时刻/星期 Embedding） |
| MissTSM | **轻** | 仅实现 MissTSM 基础版（原文有 MissTSM/MissTSMSkip/iMissTSM 三个变体）；缺少 skip connection（原文 MissTSMSkip 保留观测值 `m*x + (1-m)*out`） |

### 2.2 训练预算不足

统一 10 epoch + early stopping patience=3，联合建模方法可能欠拟合。

### 2.3 缺失率范围窄

仅测试 10% 和 30%，缺失感知方法的理论优势可能需要在更极端缺失条件（50%、70%）下才能体现。

## 3. 第二轮实验目标

1. 将 CRIB、CoIFNet、MissTSM 替换为**完整版实现**（从原始论文仓库移植）
2. 新增**高缺失率**（50%、70%）维度
3. 提升训练预算（20 epoch，patience=5）
4. 修复 SAITS+PatchTST 数值溢出
5. 精简实验维度，总量控制在 **400~500 条**

## 4. 模型替换方案

### 4.1 外部仓库获取

```bash
cd missing_ts_exp/external/
git clone https://github.com/Muyiiiii/CRIB.git
git clone https://github.com/KaiTang-eng/CoIFNet.git
git clone https://github.com/KGML-lab/MissTSM.git
```

### 4.2 CRIB 完整版移植（优先级最高）

源文件：
- `external/CRIB/TSL_models/CRIB.py`（主模型）
- `external/CRIB/TSL_models/CRIB_embedding.py`（嵌入层）
- `external/CRIB/TSL_models/CRIB_module.py`（TCN / Transformer / Encoder / PredHead）

需移植的关键特性：

| 特性 | 原版实现 | 当前简化版 |
|---|---|---|
| 输入格式 | `(B, P, N, patch_len)` patched | `(B, L, C)` 直接输入 |
| 时序嵌入 | TCNBlock（指数膨胀卷积，dilation=2^i） | 无 TCN，直接 Linear |
| 编码器 | TransformerEncoder + conv_layer 选项 | 标准 attention layers |
| 后验分布 | `MultivariateNormal(loc, diag(softplus(scale)))` | `mu + exp(0.5*logvar)*eps` |
| 双视图 | x_1（原始）+ x_2（加 1% 高斯噪声增强） | 仅单视图 |
| 一致性损失 | 两视图 enc_out 的一致性约束 | 无 |
| 预测头 | reshape `(B,N,P*d)` → 两层 Linear → `(B,pred_len,N)` | `Linear(seq_len, pred_len)` + `Linear(d_model, C)` |

适配要点：
- 数据层需支持 patching：在 `CribPipeline.forward` 中对 `(B, L, C)` 做 `unfold` → `(B, P, N, patch_len)`
- 双视图前向：训练时对同一 batch 前向两次（或拼 batch 一次前向再拆），计算一致性损失加入 `aux_loss`
- 测试时跳过第二视图

### 4.3 CoIFNet 完整版移植

源文件：
- `external/CoIFNet/model/CoIFNet.py`（SharedModule）
- `external/CoIFNet/model/CoIFNetTask.py`（任务封装）
- `external/CoIFNet/model/RevON.py`（掩码感知归一化）
- `external/CoIFNet/model/attend.py`（Flash Attention）

需移植的关键特性：

| 特性 | 原版实现 | 当前简化版 |
|---|---|---|
| Backbone 块类型 | TSBlock / LinearBlock / AttentionBlock（可配置） | 固定 MixerBlock（双 FFN） |
| Gated Attention | V 值 sigmoid 门控 + Flash Attention | 无 |
| 时间特征嵌入 | `nn.Embedding(24, embed_dim)` + `nn.Embedding(7, embed_dim)` | 无 |
| RevON | 三模式（norm / norm-fore / denorm），mask 感知 | RevIN（norm / denorm），mask 感知 |
| 任务封装 | CoIFNetTask 独立类，防评估时标签泄漏 | Pipeline 内直接计算 |

适配要点：
- 使用原版的 `AttentionBlock` 作为 inter_model，`TSBlock` 作为 intra_model（原论文默认配置）
- 时间特征嵌入可利用 `batch["x_mark"]` 中的时间信息
- RevON 的 `norm-fore` 模式用于预测段反归一化

### 4.4 MissTSM 完整版移植

源文件：
- `external/MissTSM/forecasting/misstsm_itransformer/layers/Transformer_EncDec.py`

需移植的关键特性：

| 特性 | 原版实现 | 当前简化版 |
|---|---|---|
| 变体 | MissTSM + MissTSMSkip + iMissTSM | 仅 MissTSM |
| Skip connection | `x = m * x_inp + (1-m) * out`（保留观测值） | 无（全部位置替换） |
| 注意力掩码 | 大负数偏置加到 attention score | `key_padding_mask=True` |

适配要点：
- 默认使用 **MissTSMSkip** 变体（保留观测值的 skip connection 对预测有实质影响）
- 注意力掩码方式两种实现效果等价，可保留当前 `key_padding_mask` 方式
- 保留当前的 all-missing 时间步保护（原版未做此保护）

### 4.5 SAITS+PatchTST 溢出修复

排查方向：
- SAITS 输出添加 `clamp(-10, 10)` 防止极端值传播到 PatchTST
- PatchTST 的 RevIN 统计量保护（标准差下界 clamp）
- 考虑降低 SAITS+PatchTST 的学习率或增加梯度裁剪力度

## 5. 精简实验设计

### 5.1 设计原则

1. **复用第一轮数据**：baseline（无缺失上界）、simple（线性插值）、ablation_gen（缺失类型泛化）、ablation_pred（预测模型复杂度）等已有明确结论的组不重跑
2. **聚焦核心问题**：完整版缺失感知方法 vs 两阶段最强基准
3. **新增高缺失率维度**：补充 0.5/0.7 缺失率
4. **seed 从 3 减到 2**：第一轮种子间标准差很小（多数 <0.01），2 seed 已足以确认趋势

## 7. 预期结果与分析框架

### 7.1 核心对比分析

第二轮报告将以以下对比为主线：

| 对比项 | 对照组 | 预期结论 |
|---|---|---|
| 完整版 vs 简化版 | 同方法、同配置，两轮数据对比 | 量化简化带来的性能损失 |
| 完整版 vs 两阶段最优 | CRIB/CoIFNet/MissTSM-full vs Interp+iTrans | 判断公平条件下谁更优 |
| 高缺失率下的排名 | 0.1/0.3（第一轮） vs 0.5/0.7（第二轮） | 缺失感知方法是否在高缺失率下反超 |
| 掩码消融 | 完整版 A/B/C vs 第一轮简化版 A/B/C | 掩码对完整版方法是否起正面作用 |

### 7.2 可能的场景

**场景一：完整版缺失感知方法显著提升，但仍不及两阶段**
- 说明简化确实有影响，但"简单方法即可"的结论仍然成立
- 论文叙事：缺失感知方法的附加复杂度在中低缺失率下不值得

**场景二：完整版方法在高缺失率（0.5/0.7）下反超两阶段**
- 说明缺失感知方法有适用区间，但需要极端条件才能体现
- 论文叙事：存在一个"缺失率阈值"，低于阈值用简单方法，高于阈值用缺失感知方法

**场景三：完整版方法在所有配置下均胜出或接近两阶段**
- 说明第一轮结论主要由实现简化造成
- 需要回溯修正第一轮结论

## 9. 时间估算

假设单 A800 卡，每条命令平均 3~5 分钟（20 epoch，约为第一轮 10 epoch 的 2 倍）：

- 356 条 × 4 分钟 ≈ **24 小时**（单卡）
- 8 卡并行 ≈ **3 小时**

模型移植预计 2~3 天，总周期约 **4~5 天**。
